数据操作

对于计算机来说，完成一个任务离不开三类操作：

  1. 数据的地址。
  2. 数据。
  3. 读/写数据。


当读时:

  1. 通过地址总线指定要访问的内存地址
  2. 通过控制总线指定需要读取
  3. 通过数据总线将数据传到CPU

当写时:

  1. 通过地址总线指定要访问的内存地址
  2. 通过控制总线指定需要写入
  3. 通过数据总线将数据传到内存

所以地址总线的”宽度“限制了寻址的大小，假如地址总线有16根，那么最大寻址是2的16次方。

地址送到地址总线前，是经过CPU内部的一个地址加法器处理后再送出来，主要是因为程序为了方便都是把内存进行分段管理，所以给出一个地址都是:段地址+偏移地址。

8086寄存器

每个CPU架构都会有对应的寄存器，8086寄存器大概分为三类如下：

  • 通用寄存器: 任意使用，自己控制
  • 控制寄存器: IP是存放代码下一条指令的地址，FLAG是各种状态信息
  • 段寄存器: 存放内存每个段的开始位置

程序一般为了方便会把内存进行划分，分出来的每个段都可以用段寄存器指向，当需要访问某个段的内存时，都是用:段地址+偏移地址，的方式进行访问。

例如运行代码时 CS:IP指定当前运行到的代码位置；DS:[0]指定需要访问数据段0位置的数据；SS+N进行移动栈顶指针访问栈内元素或者BP+N访问栈内元素。

FLAG内存放各种状态，某些汇编指令执行后，会使FLAG对应的标志位变化，比如cmp指令执行后，如果比较结果为真ZF标志位会置为1。

函数调用栈

当调用call指令时，可以修改IP的指向，从而把代码指向新的地址执行，改变按顺序执行的情况，当call指向的地址代码执行完后，使用ret指令，可以使IP重新指向到调用call地址的下一个地址，从而实现函数调用。

高地址

一个栈帧包含函数调用时所需要保存的“现场数据”：

  1. 函数需要接受参数，以便使用，所以先把参数压栈保存起来，等待使用。
  2. 函数调用完后需要回到调用处，所以需要把返回地址压栈，以便返回执行下一行代码。
  3. BP保存当前栈基地址，可以方便访问当前函数的参数和局部变量，当BP+N时就是访问参数，当BP-N时就是访问局部变量，所以需要把BP压栈，保存上一个函数的BP值。
  4. 当前函数所使用到的局部变量全部压栈保存，以便之后使用。
  5. 保存可能需要使用到的寄存器的值，以便函数返回后能恢复。

当函数调用完毕，会执行SP-N，将栈顶指针指向没调用前的地址，之前的栈数据是依然会存在，所以为了安全，每当申请局部变量区域会统一把区域存放CC，CC为断点信号，防止访问到垃圾数据。

一个较完整的8086汇编代码

assume cs:code, ds:data, ss:stack

; 栈段
stack segment
    db 100 dup(0)
stack ends  

; 数据段
data segment  
    db 100 dup(0) 
data ends

; 代码段
code segment
start:
    mov ax, data
    mov ds, ax    ; 手动设置ds值
    mov ax, stack 
    mov ss, ax    ; 手动设置ss的值
    
    ;----------------------------------------------------------------------
    push 1        
    push 2                                          ;<-----1. 参数
    ;----------------------------------------------------------------------


    ;----------------------------------------------------------------------
    call test                                       ;<-----2. 返回地址
    ;---------------------------------------------------------------------- 
    
    ; 程序退出
    mov ax, 4c00h
    int 21h 
    
; 返回值放ax寄存器
; 传递2个参数(放入栈中)    
test:

    ;----------------------------------------------------------------------
    push bp                                         ;<-----3. BP地址
    ;----------------------------------------------------------------------

    ;----------------------------------------------------------------------
    mov bp, sp ; 保存sp之前的值：指向bp以前的值
    sub sp, 4 ; 给局部变量”分配“4个字节的空间           ;<-----4. 保存局部变量
    ;----------------------------------------------------------------------
    
    ;----------------------------------------------------------------------
    ; 保护可能会用到的寄存器
    push si
    push di
    push bx                                         ;<-----5. 需要保存的寄存器   
    ;----------------------------------------------------------------------
     
    ;----------------------------------------------------------------------
    ; 访问栈中的参数
    mov ax, 0x1                                     
    add ax, ax                                      ;函数逻辑
    ;----------------------------------------------------------------------
    
    ; 恢复寄存器的值
    pop bx
    pop di
    pop si
                       
    mov sp, bp ; 恢复sp
    pop bp     ; 恢复bp
    add sp, 4  ; 恢复栈平衡
    
    ret 
                
code ends  

end start

8086常用汇编指令

指令名称 指令全称 作用 例子 C/C++
mov move 移动值到寄存器 mov ax 10h int a = 10
add 两个值双加 add ax 10h a = a + 10
sub subtract 两个值相减 sub ax 10h a = a - 10
call 调用一个函数 call method method()
ret return 函数返回 ret return
push 将值存到栈顶 push 10h stack.push(10)
pop 将值存从栈顶弹出 pop ax a = stack.pop()
jmp jump 改变IP的指向，从而改变代码的执行顺序 jmp 0x667788f go to 0x667788f
cmp compare 比较两个值，比较后的结果会影响FLAG的ZP标记位 cmp ax 11h if (a == 11)

一段较完整的ARM64汇编

main.c文件

int test() {
		int a = 1;
		int b = 1;
		return a + b;
}

int main(int argc, const char * argv[]) {
		int ret = test();
		return 0;
}

使用命令xcrun --sdk iphoneos clang -S -arch arm64 main.c -o main.s生成一下C语言源码转换为ARM64汇编

	.section	__TEXT,__text,regular,pure_instructions
	.build_version ios, 13, 6	sdk_version 13, 6
	.globl	_test                   ; -- Begin function test
	.p2align	2
_test:                                  ; @test
	.cfi_startproc
; %bb.0:
	sub	sp, sp, #16             ; =16
	.cfi_def_cfa_offset 16
    ;----------------------------------------------------------------------
	str	w0, [sp, #12]                                   ;<-----1. 参数
    ;----------------------------------------------------------------------

    ;----------------------------------------------------------------------
	mov	w8, #1
	str	w8, [sp, #8]
	str	w8, [sp, #4]                                    ;<-----4. 保存局部变量
	ldr	w8, [sp, #8]
	ldr	w9, [sp, #4]
    ;----------------------------------------------------------------------

    ;----------------------------------------------------------------------
	add	w0, w8, w9                                      ;函数逻辑
    ;----------------------------------------------------------------------

	add	sp, sp, #16             ; =16                   ; 恢复栈平衡
	ret
	.cfi_endproc
                                        ; -- End function
	.globl	_main                   ; -- Begin function main
	.p2align	2
_main:                                  ; @main
	.cfi_startproc
; %bb.0:
	sub	sp, sp, #48             ; =48
	stp	x29, x30, [sp, #32]     ; 16-byte Folded Spill
	add	x29, sp, #32            ; =32
	.cfi_def_cfa w29, 16
	.cfi_offset w30, -8
	.cfi_offset w29, -16
	stur	wzr, [x29, #-4]
	stur	w0, [x29, #-8]
	str	x1, [sp, #16]
	mov	w0, #1
    ;----------------------------------------------------------------------
	bl	_test                                       ;<-----2. 返回地址 和 <-----3. BP地址(将下一行命令地址存到LR，并重置PC为test函数地址)
    ;----------------------------------------------------------------------

    ;----------------------------------------------------------------------
	str	w0, [sp, #12]                               ;得到返回值
    ;----------------------------------------------------------------------
	mov	w8, #0
	mov	x0, x8
